# Word Formatter 真实回归集验证计划

## 目标
用 20-50 份真实论文样本验证 Lite/Pro 的稳定性、修复率和误伤率。

## 核心指标
- 修复覆盖率：目标规则命中后成功修复比例
- 误伤率：不应变更区域被改动比例
- 稳定性：同一输入重复运行输出一致性
- 性能：单文档处理时长

## 必含极端样本

### 1) 灾难级标点样本
- 特征：全篇混用全角/半角括号，题注中出现 `图1-1(a)`、`图1-1（a）` 混杂
- 关注：题注识别与编号修复是否误判

### 2) 跨页断裂表格样本
- 特征：长表格被分页符强制断开
- 关注：M3 三线表是否在中间页面产生错误底线/表头线

### 3) 域代码污染样本
- 特征：混用 EndNote、NoteExpress、手工残缺域代码
- 关注：`Fields.Update` 是否卡死或导致域异常

## 执行流程
1. 对每份样本做原件快照（docx + pdf + hash）。
2. 运行 Lite：
   - 导入 `lite/THU_Formatter_Lite.bas`
   - 执行 `OneClickDetectAndFix`
3. 收集输出：`*_fixed.docx`、`*_fixed.pdf`、`*_fix_log.txt`
4. 对照检查：
   - 标题层级
   - 题注和 SEQ
   - 三线表
   - 目录与交叉引用
5. 将失败样本归类：规则缺失 / 误判 / 宏环境问题 / 文档结构异常。

## 判定门槛（建议）
- 核心路径成功率 >= 95%
- 误伤率 <= 1%
- 致命失败率（中断、崩溃）<= 2%

## 输出物
- `regression_summary.xlsx`（样本编号、问题类型、结果、备注）
- `failure_cases/`（保留失败样本和日志）
- `rules_change_log.md`（规则迭代记录）
