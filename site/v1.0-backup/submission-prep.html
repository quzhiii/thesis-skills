submission prep

投稿/送审前如何 系统检查论文材料是否齐备

这一页关心的是 readiness 和 completeness，不是内容写得好不好，也不是投稿系统操作指南。目标是把最终要交出去的材料和最后一轮风险检查整理成一个清楚的 final submission packet。

6 类 送审前检查项
PASS / WARN / BLOCK readiness gate 语义
静态指引 不是 journal submission portal
适合什么时候用
  • 你已经接近提交，想系统确认还有没有明显遗漏。
  • 你需要区分内容质量检查和提交材料齐备检查。
  • 你希望最后交出去的是一个整理好的 submission packet，而不是一堆散文件。
01 / distinction

先分清两种检查

submission-prep 最容易和“最后一次读论文”混在一起。这里需要明确：内容质量检查和提交材料齐备检查不是同一回事。

content quality

内容质量检查

关注论证是否成立、语言是否通顺、章节是否完整、图表解释是否自洽。这一层更接近论文本身的写作质量与学术判断。

submission readiness

提交材料齐备检查

关注你交出去的材料是否完整、命名是否规范、附件是否齐、格式导出是否对、最后一轮风险是否已经看过。这一层更接近 readiness verification。

02 / checklist

送审前检查清单

下面这份清单是为了确认材料是否齐备，而不是代替你做投稿系统流程。

check 01

格式 / 模板

确认学校、学院、期刊或会议要求的模板和格式是否一致。

这一层通常对应 rule pack、格式检查报告和导出结果，不是靠肉眼随便扫一遍就能替代的。

check 02

摘要、关键词、标题页

确认封面、摘要、中英关键词、作者信息等基础元数据没有缺项。

这类问题不一定属于“论文内容错误”，但非常常见，而且最容易在提交前被卡住。

check 03

图表、编号、引用

确认图表编号、交叉引用、参考文献格式和引用完整性。

这一步连接 thesis-skills 最成熟的检查能力，也是最适合交给 artifact 和 gate 汇总的一层。

check 04

附件、声明、补充材料

把需要一起交的声明、附录、补充文件和外部材料单独核对。

submission-prep 的重点之一就是把“论文正文之外但必须一起交”的东西整理成一个清楚的包。

check 05

文件命名与导出格式

确认最终 PDF、源文件、附录、压缩包和命名规则符合要求。

这一步最像“交付包装”，不是论文内容本身，但决定你交出去的 submission packet 是否整洁可靠。

check 06

最后一轮风险检查

在真正送审前，再看一轮 blockers、warnings 和 unresolved review debt。

如果你的目标是正式提交，这一步就该切到更严格的 `submission-prep` 语义，而不是继续停留在宽松的 advisor-handoff 语义上。

03 / packet

final submission packet 里应该有什么

最终整理出来的不是“一个文件夹里随便堆几个版本”，而是一个可以被快速检查、快速交付的 final submission packet。

01

最终 PDF / 主文件

确保最终导出结果与准备提交的版本一致，不要混入旧稿或试验性版本。

02

readiness artifact / checklist

保留最后一轮 PASS / WARN / BLOCK 结果，以及你自己用来过提交清单的最终版本。

03

附件与声明

把声明、补充材料、附录或需要额外交付的文件放进同一个 submission packet 视图里。

04 / warning

这不是投稿系统，也不是送审门户

submission-prep 页面只教你怎样把材料和风险检查整理清楚，不会引入上传、登录、状态跟踪、分配任务或自动提交流程。

avoid

不要把它误解成什么

  • 不是 journal submission portal。
  • 不是学校送审系统镜像。
  • 不是一个“帮你自动完成提交”的工作流自动化平台。
related

更贴近 thesis-skills 的用法

更适合把它看成一个 final-readiness scenario guide：告诉你在真正交出去之前，哪些 artifact、哪些 checklist、哪些风险判断应该先收齐。
