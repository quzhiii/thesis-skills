# V0.8 未来功能清单

以下功能被**有意排除**在 V0.7 落地页之外，但可能在后续版本中出现。

## 已规划的功能

### 提交级导出 (submission-friendly export)
- 当前 v0.7 仅支持 review-friendly 导出
- v0.8 可能增加更严格的 submission-friendly 导出模式
- 需要更完整的格式保持和更少的降级

### 扩展规则包覆盖
- 更多高校的自定义规则包
- 期刊投稿专用规则包
- 学科特定规则（理工科、人文社科等）

### 改进的审阅闭环
- 支持更多批注格式导入
- 审阅历史的版本对比
- 多轮 revision 的状态追踪

## 考虑中的功能

### 轻量级 Web 界面
- 一个极简的 HTML 报告查看器
- 在浏览器中查看检查报告和修复建议
- **不是** 完整的 Web 编辑器或 IDE

### 增强的编译诊断
- 支持更多 LaTeX 引擎的日志解析
- 与常见 TeX 发行版的更好集成
- 编译问题的自动修复建议

### 扩展文档
- 视频教程
- 更多示例项目
- 规则包编写指南

## 明确不在路线图上的功能

以下功能**不会**在可预见的未来实现，以保持项目的定位和范围：

- ❌ AI 论文写作助手（自动生成内容）
- ❌ 通用 LaTeX 到 Word 的完美转换器
- ❌ 自动理解导师自然语言批注
- ❌ 多用户实时协作平台
- ❌ 替代 Overleaf / TeXstudio 的完整 IDE
- ❌ 一键完成论文提交

## 设计约束

v0.8 及以后的版本将继续遵循以下原则：

1. **有边界的工作流**：每个功能都有明确的输入、输出和限制
2. **显式报告**：不支持的功能和降级都被明确报告
3. **不替代人工判断**：高判断成本的决策仍由人做最终决定
4. **开源可审计**：核心逻辑保持开源和可自定义
