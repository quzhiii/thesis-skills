01 / pain-points

写论文时，这些处境是否熟悉

01

导师只审 Word，论文是 LaTeX

导出格式不兼容，手动转换容易丢格式，批注无法对应回源文件。

02

LaTeX 编译报错看不懂

原始日志几百行，真正需要修的错误埋在中间，定位靠运气。

03

修改意见分散，难以追踪

批注、邮件、口头反馈散落各处，新一轮修改时不知道哪些已处理。

04

提交前心里没底

不确定引用是否完整、格式是否合规、有没有遗漏的章节要求。

02 / capabilities

四个核心检查能力

每项检查生成独立 JSON 报告，列出具体发现和位置，不静默修改源文件。

01

引用完整性检查

对照 .bib 扫描 \cite{key}，报告缺失引用、孤立 bib 条目、编号顺序异常，附行号。

02

CJK 语言风格检查

标记中英文标点混用、弱语气词（「众所周知」「不难看出」等），逐条列出位置。

03

图表结构验证

找出缺 label 或 caption 的 figure / table，以及 longtable 续页标记缺失。

04

Word → LaTeX 迁移

章节拆分 + 引用格式转换，迁移结果同步经过引用和格式检查，生成映射报告。

03 / artifacts

每一步都有明确的产物

检查器不直接修改源文件，生成结构化 JSON 报告；修复器读取报告后生成补丁，支持 dry-run 预览。

compile-findings.json
{
  "severity": "error",
  "source": "thesis.tex:214",
  "message": "Undefined control sequence",
  "hint": "检查 \\newcommand 定义"
}

编译错误变成可读诊断。

review-todo.json
{
  "id": "REV-003",
  "status": "pending",
  "source": "导师批注 P.12",
  "category": "论述深度",
  "action": "补充实验对比"
}

返修意见变成待办项。

readiness-gate.json
{
  "verdict": "WARN",
  "blockers": [],
  "warnings": ["3 个深度语言问题需审阅"],
  "next_action": "优先处理 warnings"
}

提交状态有明确裁决。

04 / trustworthy

有边界、可检查、不夸大

有边界的工作流

每个功能有明确的输入、输出和限制。不支持的构造显式报告，不静默降级。

显式报告

检查器生成 JSON，不直接修改源文件。修复器读报告生成补丁，支持 dry-run。

产物可检查

每步输出人类可读的 artifact，你可以打开 JSON 查看具体发现。

没有魔法

不声称能自动理解导师意见或智能修复所有问题。含糊的修改由人做最终决定。

局限性被暴露

导出限制、检查覆盖范围，所有边界在产物中显式报告，不藏着。

开源可审计

MIT 协议，所有规则、检查逻辑和 YAML 规则包都可以查看和自定义。

05 / quick start

一条命令跑完所有检查

调用 run_check_once.py，依次执行各检查模块，读取 YAML 规则配置，输出 JSON 报告。

tsinghua-thesis

清华 thuthesis，零配置

python run_check_once.py \
  --project-root "../thuthesis-v7.6.0" \
  --ruleset tsinghua-thesis
note--skip-compile 可跳过编译，加快迭代速度
custom rules

其他院校，填四个 YAML

python run_check_once.py \
  --project-root "../your-thesis" \
  --ruleset my-university
note在 90-rules/packs/ 下创建规则包，维护 pack.yaml / rules.yaml / mappings.yaml
06 / scope

v1.0 已有什么，还没有什么

帮你判断 Thesis Skills 是否适合你当前的阶段。

released

v1.0 已包含

  • 引用完整性检查（缺失、孤立、重复）
  • CJK 语言风格检查（标点、弱语气词）
  • 图表结构验证（label / caption / 交叉引用）
  • Word → LaTeX 章节拆分 + 引用迁移
  • Zotero / EndNote 导出 .bib 质量检查
  • 预提交 readiness gate（PASS / WARN / BLOCK）
  • 答辩前整理 artifact（大纲、要点、图表、讲稿备注）
  • 可自定义 YAML 规则包、lint 和 scorecard
planned

尚未支持 / 不承诺

  • 图形界面或 Web 编辑器
  • 自动理解导师自然语言批注
  • AI 生成或重写论文内容
  • 一键完成论文提交
