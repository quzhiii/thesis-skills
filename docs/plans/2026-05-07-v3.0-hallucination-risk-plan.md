# Citation Integrity V3.0 Hallucination Risk Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a deterministic V3.0 hallucination-risk layer that converts local bibliography metadata plus V2.0 external verification evidence into per-reference risk scores, a high-risk CSV, and demo projects for false metadata, fabricated references, and unsupported Chinese references.

**Architecture:** Keep V2.0 external verification as the evidence source and do not add LLMs or automatic citation rewriting. Add a small risk-scoring module under `core/citation_integrity/`, a dedicated CLI under `19-check-hallucination-risk/`, and report writers that produce JSON plus `high-risk-references.csv`. The readiness gate may consume the V3 report later as advisory evidence, but this plan keeps the first implementation focused on deterministic reports and demos.

**Tech Stack:** Python standard library only, existing `BibEntry` and external verification report shapes, pytest/unittest, CSV via `csv`, JSON via `json`, no live network in tests.

---

## Scope Boundaries

### In scope for V3.0

- Deterministic `hallucination_risk_score` per bibliography entry.
- Risk labels: `PASS`, `WARN`, `REVIEW`, `HIGH_RISK`, `UNSUPPORTED`.
- `reports/hallucination-risk-report.json`.
- `reports/high-risk-references.csv`.
- Chinese reference detection that returns `UNSUPPORTED`, not `HIGH_RISK`.
- Demo projects for:
  - real reference with wrong metadata fields
  - likely fabricated AI reference
  - Chinese reference that cannot be automatically verified
- Tests that use mocked or local fixture evidence only.

### Explicitly out of scope

- No LLM usage.
- No automatic citation or bibliography rewrite.
- No new live-network tests.
- No PubMed or DBLP provider integration in this phase.
- No claim-citation support triage. That remains V4.0.
- No candidate-reference recommendation. That remains V5.0.

## Risk Scoring Rules

Use deterministic additive scoring capped at `1.0`.

### Unsupported first

If a reference appears to be Chinese-language or outside supported provider coverage, return:

```text
risk_label = UNSUPPORTED
hallucination_risk_score = 0.0
```

Signals:

- CJK characters in `title`, `journal`, `booktitle`, or `publisher`.
- Entry type is `mastersthesis`, `phdthesis`, `misc`, or `unpublished` and no DOI exists.
- External verification report marks all providers unavailable and the title contains CJK.

### Supported English references

Suggested scoring:

| Evidence | Score contribution |
|---|---:|
| External match status is `MATCH` with DOI exact match | 0.0 |
| External match status is `MATCH` by title only | 0.15 |
| External match status is `REVIEW` | 0.35 |
| External match status is `NO_CANDIDATE` | 0.75 |
| External match status is `UNAVAILABLE` | 0.25 |
| No provider found a candidate | +0.20 |
| DOI mismatch | +0.25 |
| Title similarity below 0.70 | +0.20 |
| Year mismatch | +0.10 |
| Venue mismatch | +0.10 |

Map score to labels:

| Score | Label |
|---:|---|
| `0.00 <= score < 0.25` | `PASS` |
| `0.25 <= score < 0.50` | `WARN` |
| `0.50 <= score < 0.75` | `REVIEW` |
| `0.75 <= score <= 1.00` | `HIGH_RISK` |

Do not output text like "this is fake". Use conservative language:

```text
HIGH_RISK: no credible match was found in the enabled external databases. Manually verify DOI, journal website, database record, or source PDF.
```

## Implementation Notes

- V3.0 should consume V2.0 evidence. Do not make it call CrossRef, OpenAlex, or Semantic Scholar directly.
- If `external-verification-report.json` is missing, the CLI should still write a report with conservative `REVIEW` or `UNSUPPORTED` entries.
- `UNSUPPORTED` means "cannot be automatically judged by enabled evidence," not "safe".
- `HIGH_RISK` means "manual verification strongly recommended," not "fake".
- Keep network-related tests mocked or fixture-only.
- Keep scoring thresholds in code constants so future calibration is easy.
- Do not introduce LLM prompts, API calls, or generated citation suggestions.
