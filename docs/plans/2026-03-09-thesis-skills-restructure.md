# Thesis Skills 技术路线图

## 项目演进历史

### v0.1.0 - 初始版本（2025 Q1）

**核心目标**：解决 Word → LaTeX 转换过程中的基础问题

**主要功能**：
- 基础的 Python + Skills 架构
- 简单的检查器框架
- 手动规则定义

**局限性**：
- 检查和修复逻辑耦合
- 规则硬编码在代码中
- 缺乏引用映射机制
- 无法处理 Zotero 引用

### v0.2.0 - 架构重构（2025 Q2）

**核心目标**：模块化设计，分离检查和修复逻辑

**主要改进**：
- ✅ 分离 `check` 和 `fix` 模块
- ✅ 引入 YAML 规则包系统
- ✅ 创建 Starter Pack（university-generic、journal-generic）
- ✅ 添加 tsinghua-thesis 示例规则包
- ✅ 实现规则加载器（core/rules.py）
- ✅ 实现 YAML 解析器（core/yamlish.py）
- ✅ 添加项目发现功能（core/project.py）
- ✅ 创建一键运行器（run_check_once.py、run_fix_cycle.py）
- ✅ 添加接入指南（adapters/intake/README.md）

**架构变化**：
```
旧架构：
[Skills] → [检查+修复耦合] → [输出]

新架构：
[Skills] → [检查器] → [JSON 报告] → [修复器] → [输出]
                ↓
            [YAML 规则包]
```

### v0.3.0 - Zotero 同步（2025 Q3）**当前版本**

**核心目标**：解决 Word → LaTeX 转换时的文献引用管理问题

**主要功能**：
- ✅ Zotero 引用提取（core/zotero_extract.py）
- ✅ 引用映射管理（core/citation_mapping.py）
- ✅ Word → LaTeX 同步（00-bib-zotero/sync_from_word.py）
- ✅ Citation Lock 机制（citation-lock.tex）
- ✅ 增量更新支持
- ✅ BibTeX 质量检查

**核心创新**：

**引用映射机制**：
```json
{
  "WuZeXinZhongYiYouShiBingZhongNaRuDRGDIPFuFeiGuiFanFenXia": "ref001",
  "ZhangSan2020SomeTitle": "ref002"
}
```

**Citation Lock 机制**：
```latex
% citation-lock.tex
\nocite{ref001,ref002,ref003,...}
```

这确保：
- 删除引用时不会改变后续编号
- 所有 bib 条目都会出现在参考文献中
- 引用编号始终保持稳定

**工作流程**：
```
Word (Zotero) → [提取 CSL_CITATION] → [比较差异]
                                          ↓
                                    [更新映射]
                                          ↓
                                    [生成 citation-lock.tex]
                                          ↓
                                    [LaTeX 编译]
```

## v0.2 vs v0.3 详细对比与效率分析

| 对比维度 | v0.2 | v0.3 | 效率提升 |
